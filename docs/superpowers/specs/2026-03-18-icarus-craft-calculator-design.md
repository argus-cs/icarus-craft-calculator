# Icarus Crafting Calculator ‚Äî Design Spec

## Overview

A mobile-first web app for looking up crafting recipes from the game Icarus. Users can browse items by category, view crafting materials and stations, and adjust quantities to calculate total material requirements. The app works offline as a PWA.

## Goals

- Provide a fast, searchable list of craftable items from Icarus
- Show item details: materials, quantities, and crafting stations
- Quantity multiplier that recalculates all material requirements
- Drill into materials recursively (material ‚Üí sub-materials ‚Üí base resources)
- Work offline on mobile devices (PWA)
- Easy to update when the game changes (re-run scraper)

## Scope

### In Scope

- **Item categories:** Tools, Constructions, Deployables, Resources
- **Resources:** Base materials (wood, stone, ore, etc.) and intermediate materials (ingots, composites, etc.) as a full browsable category
- **Scraper:** Node.js script that extracts data from the Icarus Fandom Wiki
- **Web app:** Vue 3 + Nuxt 3 SSG with Tailwind CSS
- **PWA:** Installable, works offline
- **Deploy:** GitHub Pages via GitHub Actions

### Out of Scope

- Backend / API server
- User accounts or saved builds
- Items outside the 4 browsable categories (weapons, armor, consumables)
- Automatic scraping schedule (manual execution only)

## Data Model

Single JSON file generated by the scraper at `public/data/items.json`:

```jsonc
{
  "version": "1.0.0",
  "generatedAt": "2026-03-18T12:00:00Z",
  "items": [
    {
      "id": "steel-pickaxe",
      "name": "Steel Pickaxe",
      "category": "tools",           // "tools" | "constructions" | "deployables" | "resources"
      "tier": "tier-3",
      "icon": "/icons/steel-pickaxe.png",
      "craftedAt": ["machining-bench"],
      "materials": [
        { "itemId": "steel-ingot", "quantity": 8 },
        { "itemId": "wood", "quantity": 4 }
      ]
    },
    {
      "id": "steel-ingot",
      "name": "Steel Ingot",
      "category": "resources",       // intermediate craftable material
      "tier": "tier-3",
      "icon": "/icons/steel-ingot.png",
      "craftedAt": ["concrete-furnace"],
      "materials": [
        { "itemId": "iron-ore", "quantity": 2 },
        { "itemId": "carbon-ore", "quantity": 1 }
      ]
    },
    {
      "id": "wood",
      "name": "Wood",
      "category": "resources",       // base material (leaf node)
      "tier": null,
      "icon": "/icons/wood.png",
      "craftedAt": [],
      "materials": []
    }
  ],
  "stations": [
    {
      "id": "machining-bench",
      "name": "Machining Bench",
      "category": "deployables",
      "tier": "tier-2"
    }
  ]
}
```

**Key decisions:**
- **Flat structure with ID references** ‚Äî materials point to other items via `itemId`, stations via ID
- **Resources category** ‚Äî base materials (`wood`, `stone`, `fiber`) and intermediate materials (`steel-ingot`, `electronics`) are included with `category: "resources"`. They are a full browsable category in the UI, just like Tools, Constructions, and Deployables.
- **Leaf detection** ‚Äî items with `materials: []` are base resources (recursion stops). Items with `craftedAt: []` are gathered/mined, not crafted.
- **Stations as items** ‚Äî many stations are craftable deployables, so they exist both in `items` and `stations`. The `stations` array is a quick lookup.
- **Quantity multiplication** ‚Äî computed client-side: `material.quantity √ó desiredQuantity`
- **Versioning** ‚Äî `version` and `generatedAt` fields allow the app to display data freshness

## Architecture

Monolith approach: scraper script and Nuxt app in the same repository. The scraper generates static JSON that is committed to the repo. CI only runs `nuxt generate`.

```
User runs scrape.ts ‚Üí generates items.json (committed) ‚Üí push ‚Üí CI runs nuxt generate ‚Üí deploys to GitHub Pages
```

## Scraper Design

**Source:** Icarus Fandom Wiki (https://icarus.fandom.com/)

**Process:**
1. Fetch category pages for Tools, Constructions, Deployables, and Resources
2. Parse HTML tables with Cheerio to extract item data
3. For each item: name, tier, materials (name + quantity), crafting stations
4. Normalize IDs (slugify names)
6. Validate output with Zod schema ‚Äî fail with clear error if no items found
7. Write `public/data/items.json` with `version` and `generatedAt` fields
8. Download icons to `public/icons/` with a placeholder fallback for missing ones

**Validation & error handling:**
- Zod schema validates the generated JSON structure before writing
- Warnings logged for items that fail to parse (skipped, not fatal)
- Fatal error if zero items are scraped (prevents deploying empty data)
- Summary log at the end: items scraped per category, warnings count

**Execution:**
```bash
npx tsx scripts/scrape.ts
```

**Structure:**
```
scripts/
  scrape.ts               # Entry point ‚Äî orchestrates scraping
  scrapers/
    tools.ts              # Tools category scraper
    constructions.ts      # Constructions category scraper
    deployables.ts        # Deployables category scraper
    resources.ts          # Resources category scraper (base and intermediate materials)
  utils/
    parser.ts             # Common HTML parsing functions
    writer.ts             # Validates and writes final JSON
    schema.ts             # Zod schemas for validation
```

## UI Design

### Theme

- Dark mode: `#0f172a` background, `#1e293b` cards, `#334155` borders
- Accent: amber `#fbbf24` (evokes Icarus fire theme)
- Max-width 480px centered ‚Äî works on desktop too
- Placeholder icon: generic item silhouette used when icon is missing

### Navigation Flow

```
List (index.vue) ‚Üí Item Detail (items/[id].vue) ‚Üí Material Drawer (overlay)
```

### Screen 1: Item List (`pages/index.vue`)

- **Header:** App title + search input
- **Category tabs:** Tools | Constructions | Deployables | Resources (filters the list)
- **Item cards:** Icon, name, tier badge, material count, chevron
- Search filters by item name in real-time

### Screen 2: Item Detail (`pages/items/[id].vue`)

- **Header:** Back arrow + item name + tier badge
- **Item info:** Large icon, name, category, tier
- **Quantity input:** +/‚àí buttons with value in amber. Default: 1
- **Materials list:** Each row shows icon, name, calculated quantity (base √ó multiplier). Rows are clickable (open drawer for craftable materials)
- **Crafting station:** Badge with station name and tier
- **Base resources:** For items with no recipe (`materials: []`, `craftedAt: []`), the detail page shows item info with a "Base resource ‚Äî gathered in the world" message. The quantity input and materials list sections are hidden.

### Screen 3: Material Drawer (`MaterialDrawer.vue`)

- **Bottom sheet** with drag handle for swipe-to-dismiss
- **Semi-transparent backdrop**
- **Content:** Material name/tier, quantity (inherited from calculation), sub-materials list, crafting station
- **Recursive navigation:** Clicking a sub-material replaces the drawer content with a slide-left animation. An internal navigation stack keeps history. A back button (‚Üê arrow) at the top lets the user go back to the previous material. Swiping down dismisses the entire drawer.
- **Base resources:** When a material has no sub-materials (`materials: []`), it is displayed as a leaf ‚Äî no chevron, not clickable.

## Composables

| Composable | Responsibility |
|-----------|---------------|
| `useItems()` | Loads items.json, provides search and category filtering. Exposes `findById()` for material lookups including resources. |
| `useCraftCalculator()` | Computes material quantities for a given item √ó quantity. Returns first-level materials with computed amounts (base √ó multiplier). Does NOT recursively flatten ‚Äî each drawer level computes its own materials independently. |
| `useStations()` | Station lookup by ID |

**`useCraftCalculator` behavior:**
- Input: item ID + desired quantity
- Output: array of `{ item, computedQuantity }` for direct materials
- Each material's quantity = `material.quantity √ó desiredQuantity`
- Recursion is handled by the UI: when a material drawer opens, it calls `useCraftCalculator` again with the sub-material and its computed quantity

## Deploy

- **GitHub Pages** via GitHub Actions
- `items.json` and `icons/` are committed to the repo ‚Äî CI does NOT run the scraper
- Workflow triggers on push to `main`
- Runs `nuxt generate` and deploys the `.output/public` directory
- `baseURL` configured in `nuxt.config.ts` for GitHub Pages path
- Node version specified in workflow

## Testing Strategy

- **Scraper parser:** Unit tests with Vitest for `parser.ts` functions against HTML fixtures
- **Craft calculator:** Unit tests with Vitest for `useCraftCalculator` ‚Äî verifies quantity multiplication and edge cases (zero quantity, missing materials)
- **Schema validation:** Snapshot test of scraper output against Zod schema
- **Components:** Visual testing in browser during development
- **PWA:** Manual test of install and offline mode on mobile device
